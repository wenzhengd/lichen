# Derivations

## 1. Toggling-Frame Generator

The physical dephasing operator is `\sum_a Z_a` in the lab frame. Under the ideal Clifford history, the noisy segment sees

$$
A_j = G_j^\dagger \Big(\sum_{a=1}^n Z_a\Big) G_j.
$$

Because `G_j` is Clifford, each `A_j` is a signed Pauli sum.
This is the first reduction: the coherent environment is still exact, but it is expressed in the Pauli basis.

In practice this means the environment model is no longer tied to a specific
device-level Hamiltonian at runtime.  The only circuit-dependent object that
must be propagated is the Clifford frame history, which turns the fixed lab
operator into a segment-specific signed Pauli generator.

## 2. Segment Unitary

For a segment of duration `\Delta_j`, the conditional unitary is

$$
U_j(\xi) = e^{-i \xi \Delta_j A_j}.
$$

If the ideal circuit is partitioned into a block `b`, the exact conditional block unitary is

$$
U_b(\xi) = \prod_{j \in b} e^{-i \xi \Delta_j A_j}.
$$

This is the object the blockwise construction preserves exactly before projection.

The order matters: the circuit history determines the sign and support of each
`A_j`, so the block unitary is not generally reducible to a single sum of
commuting terms unless the block structure itself makes that simplification
possible.

## 3. Pauli Expansion of a Block

For a fixed `\xi`, the block unitary can be expanded in the Pauli basis:

$$
U_b(\xi) = \sum_P u_{b,P}(\xi) P.
$$

The simulator-relevant block probabilities are then

$$
q_{b,P}(\xi) = |u_{b,P}(\xi)|^2,
$$

with

$$
 u_{b,P}(\xi) = 2^{-n} \operatorname{Tr}[P U_b(\xi)].
$$

`lichen` computes these probabilities through sparse Pauli-amplitude propagation on the generated block support, rather than by full retained-diagonal reconstruction over all `4^n` Paulis.

The practical reason for this choice is scaling:

- `pro` asks for the full retained-diagonal spectrum and then reconstructs an
  effective global Pauli channel;
- `max` samples segmentwise, which is cheap but can erase cancellation too
  early;
- `ultra` computes the exact short-block Pauli amplitudes only on the support
  generated by the block, then exports a sparse block-fault object.

That is the derivation path used by the package:

1. build the Clifford frame history,
2. form the signed segment generators,
3. propagate exactly inside a short block,
4. project only at the block boundary,
5. sample or truncate the resulting sparse block channel.

## 4. Exact-to-Approximate Boundary

The exact hidden-memory process is still not directly simulable at large scale because it retains coherent off-diagonal structure across the whole circuit.
The approximation boundary is therefore placed at the block edge:

- exact within a short block,
- Pauli projection at the block boundary,
- hidden memory shared across all blocks in the shot.

That is the core derivation behind the current `lichen` implementation.

## 5. Relationship To The Package Modules

- `src/lichen/frame_tracker.py` builds the cumulative Clifford frames.
- `src/lichen/segment_generators.py` turns those frames into signed segment
  generators.
- `src/lichen/block_probability_search.py` performs the exact short-block
  Pauli-amplitude propagation.
- `src/lichen/block_fault_export.py` converts the exact block support into a
  sparse simulator-facing object.
- `src/lichen/block_sampler.py` samples one block fault per block using the
  shared hidden memory.

This division keeps the derivation readable while matching the code structure
one-to-one.
